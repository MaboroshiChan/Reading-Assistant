# Subsentence Analysis Prompt (v1)

You analyse a focused fragment of a sentence (a span inside a sentence) and
explain its micro-structure. Produce a JSON object that follows the contract
used by the Reading Assistant client for `SubSentenceAnalysis`.

Always respond with **plain JSON** and no commentary. The shape must be:

```json
{
  "analysis": {
    "sentenceId": "string",
    "text": "string",
    "units": [
      {
        "id": "unit-1",
        "text": "string",
        "role": "subject|predicate|object|complement|modifier|connector|clause|phrase|token",
        "semantics": "cause|result|condition|concession|purpose|contrast|transition|example|definition|emphasis|topic|comment|time|location|manner|evaluation|attribution|reporting|modality|none",
        "semRole": "Agent|Patient|Theme|Experiencer|Instrument|Goal|Source|Location|Time|Manner|Cause|Condition|None",
        "confidence": 0.0,
        "source": "model",
        "children": [],
        "clause": { "... optional nested SubSentenceAnalysis ..." },
        "meta": {},
        "viewHint": {
          "variant": "blue|green|yellow|gray",
          "collapsed": true,
          "label": "string",
          "order": 0
        }
      }
    ],
    "backbone": {
      "subjectId": "unit-1",
      "predicateId": "unit-2",
      "objectId": "unit-3"

```

Guidelines:
- Use the provided sentence fragment verbatim in `analysis.text`.
- Generate concise, unique `id` values per unit (e.g., `u1`, `u2a`).
- Only include fields that you can justify. Omit optional keys instead of
  inventing data.
- Prefer shallow trees: 3–6 top-level units covering the fragment without
  overlap. Nest only when a unit is clearly a clause or phrase bundle.
- Populate `role`, `semantics`, and `semRole` with the canon values listed
  above. When unknown, omit the property (do **not** invent alternatives).
- Set `source` to `"model"` for generated units and supply a `confidence`
  between 0 and 1.
- Provide `backbone.subjectId` / `predicateId` / `objectId` when the roles are
  present.
- If you supply a nested `clause`, it must itself follow the same analysis
  structure.
- Do not include span offsets; focus on textual units and hierarchy.
- Return valid JSON only — no markdown fences, trailing commas, or comments.
